HTML

  • The head element contains information about the webpage
    • The charset element sets our character encoding
    • The http-equiv element sets and HTTP header for content
    • The viewport element sets the default view for our webpage
  • The body element represents the visible content shown to the user
  • The title element defines the title for a webpage. This element is required and is extremely important for SEO (Search Engine Optimisation)
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set, description, keywords, author and viewport
    • The user wont see elements in the head or meta elements
  • The script element embeds client-side scripts in the HTML document

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Declarations contain two important components: the CSS property we want to apply and the value of that property
    • e.g. the declaration color: blue; includes the CSS property color and the value for that property, blue
  • Multiple selectors, such as header and footer, can be combined into a single CSS rule with a comma, allowing the declarations to affect both selectors
    • This helps us write Do Not Repeat Yourself (DRY) code
  • text-align defines the alignment of the affected text
  • font-size defines the size of the affected text
  • The css style sheet must be added to the HTML file using a link element that preceeds the title element under the head section
    • rel specifies the relationship between the current document and the linked document resource
    • href specifies the location of the external resource

Git

  • git status checks what branch we are currently in
  • git checkout main switches to the main branch
  • git checkout -b branch-name creates a new branch and switches to it
  • git commit -m "add descriptive message here" commits content with a descriptive message attacthed describing what was changed
  • git push Transmit local branch commits to the remote repository branch
  • git pull fetch and merge any commits from the tracking remote branch

JavaScript

  • It is good practice to end each line of code with a ;
  • In JavaScript, arrays start at 0
  • A variable is a named container that allows us to store data in our code
    • var is used to declare a new variable
    • This should be given an appropriate name that fits its use case, instead of something random
  • function is used similarly to var to create a function
    • This function must be called in the code in order to be executed
  • if and else if statements are conditionals that call different blocks of code depending on what condition has been met
  • for loops execute a block of code a number of times, such as for the length of an array
  • Control flow is the order in which a computer executes code in a script
  • console.log('message') is used to display a message in the developer console
